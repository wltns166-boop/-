# 구글드라이브 자동 저장 연동 방법

고객등록 · DB배정현황 · 병력정리 · 보장분석에서 **처리한 결과를 팀 공용 구글드라이브(구글 시트)에 자동 저장**하고,
새 DB가 들어와 데이터가 바뀔 때마다 **자동으로 최신화**합니다.

- 저장 위치: **사장님(관리자) 구글드라이브 한 곳** — 모든 팀원의 작업이 여기로 모입니다.
- 파일 형식: **구글 시트 1개**(`TEAM TOPS 데이터`) 안에 탭(시트)으로 구분
  - `고객등록` · `DB배정현황` · `병력정리` · `보장분석`
- 갱신 방식: 변경될 때마다 **전체 최신 목록으로 덮어쓰기**(중복 없음, 삭제·수정도 반영)
- 팀원은 **구글 로그인 불필요** (관리자 계정으로 동작하는 웹앱이 대신 저장)

---

## 1단계 — Apps Script 웹앱 만들기 (사장님 구글계정으로, 1회만)

1. 브라우저에서 **script.google.com** 접속 → **[새 프로젝트]**
2. 화면의 기본 코드를 전부 지우고, 저장소의 **`google-drive-sync.gs`** 파일 내용을 **전체 복사해서 붙여넣기** → 저장(💾)
3. 오른쪽 위 **[배포] → [새 배포]**
4. **[유형 선택](톱니바퀴) → [웹 앱]**
   - 설명: `drive sync`
   - **실행 계정: 나**
   - **액세스 권한: 모든 사용자**  ← 꼭 "모든 사용자"
5. **[배포]** → 권한 승인(본인 구글계정 허용. "안전하지 않음" 경고가 나오면 *고급 → 이동* 후 허용)
6. 표시되는 **웹 앱 URL** ( `https://script.google.com/macros/s/.../exec` ) **복사**

> 처음 데이터가 들어오면 사장님 드라이브에 **`TEAM TOPS 자료`** 폴더와 **`TEAM TOPS 데이터`** 시트가 자동으로 생깁니다.

---

## 2단계 — 인트라넷에 URL 연결

복사한 **웹 앱 URL을 저에게 알려주세요.** `index.html` 에 넣어 배포하면 끝입니다.

(직접 넣고 싶다면: `index.html` 상단의 `window.DRIVE_SYNC_URL = "";` 따옴표 안에 URL을 넣으세요.
임시로 브라우저 콘솔에서 `localStorage.setItem('tops_drive_sync_url','붙여넣기')` 로도 테스트할 수 있습니다.)

---

## 동작 방식

| 메뉴 | 저장되는 탭 | 갱신 시점 |
|---|---|---|
| 고객 등록 | `고객등록` | 고객 추가·수정·삭제 시마다 전체 최신화 |
| DB 배정현황 | `DB배정현황` | 배정 추가·변경 시마다 전체 최신화 |
| 병력정리(목록 저장) | `병력정리` | 결과 저장 시마다 추가·최신화 |
| 보장분석 작성 | `보장분석` | 작성 저장 시마다 추가·최신화 |

- 연결 직후, 기존에 쌓여 있던 데이터는 **최초 1회 자동으로 전부 업로드**됩니다.
- 언제든 전체를 다시 올리려면 브라우저 콘솔에서 `driveSyncAll()` 을 실행하면 됩니다.

### 고객별 폴더 저장 (NEW)
위 중앙 시트와 **별도로**, 같은 데이터를 **고객 한 명마다 폴더로 나눠서도** 저장합니다.

```
TEAM TOPS 자료 /
  └ {팀원 이름} /                    ← 담당자
      └ {고객 이름} /                ← 코드인원(고객)
          ├ 고객등록 /   → "000 고객등록 파일"   (구글시트)
          ├ DB배정현황 / → "000 DB배정 리스트"   (구글시트)
          ├ 병력정리 /   → "000"                 (구글문서)
          └ 보장분석 /   → "000"                 (구글문서)
```

- `000` 자리에는 **고객 성함**이 들어갑니다.
- 고객등록·DB배정은 **항목/내용 2열 구글시트**, 병력정리·보장분석은 **구글문서**로 저장됩니다.
- 담당자(팀원) 폴더 기준: 병력정리·보장분석·DB배정은 기록된 **담당자**, 고객등록은 **등록한 팀원**입니다.
- 같은 고객의 내용이 바뀌면 같은 파일을 **덮어쓰기**합니다(내용 변화가 없으면 다시 만들지 않음).
- 연결 직후 기존 데이터는 **최초 1회 자동으로 폴더에 백필**됩니다.
  다시 전부 올리려면 콘솔에서 `driveSaveAllCustFiles()` 를 실행하세요.

### 보험금 청구 파일 (PDF) 저장
청구관리 → 청구신청에서 **보험금 청구를 신청하면**, 생성된 청구 PDF가 같은 구글드라이브에
**팀원별 폴더**로 자동 저장됩니다.

```
TEAM TOPS 자료 /
  └ 보험금청구 /
      └ {신청한 팀원 이름} /
          └ 000님 0000년00월00일 보험금청구 /     ← 신청 1건 = 폴더 1개
              ├ 000님 0000년00월00일 보험금청구 (A보험사).pdf
              └ 000님 0000년00월00일 보험금청구 (B보험사).pdf
```

- 보험사가 여러 곳이면 폴더 안에 보험사별 PDF가 들어갑니다.
- 신청과 동시에 **총무님께 알림**이 가고, **업무관리 → 보험금 청구** 목록에도 자동 등록됩니다.

## 참고 / 주의
- `고객등록` 탭에는 주민번호 등 민감정보가 포함될 수 있습니다. 해당 구글 시트의 **공유 범위를 반드시 제한**(관리자 본인 또는 팀원 일부)하세요.
- 첨부파일(이미지·PDF의 base64)은 시트에 넣지 않고 `(첨부)` 로 표시합니다.
- URL 미설정 상태에서는 기능이 조용히 꺼져 있어 기존 동작에 영향이 없습니다.
