# 보안 전환 가이드 (TEAM TOPS 인트라넷)

지금 상태: 누구나 데이터에 접근 가능(인증 없음) + 코드/저장소가 공개 → **고객정보 유출 위험**.
목표: **로그인한 사람만** 데이터 접근. 아래 순서를 **그대로** 따라 하세요.

> ⚠️ 코드 수정(이 브랜치)만으로는 노출이 멈추지 않습니다. **3번(규칙 게시)** 을 해야 실제로 차단됩니다.

---

## 0단계 — 지금 바로 (노출 줄이기, 5분)

- **GitHub 저장소 3개를 비공개로**: `-`, `team-tops`, `parkjisoon`
  - 각 저장소 → **Settings → General → 맨 아래 Danger Zone → Change visibility → Private**
- `team-tops` 의 **Pages 끄기**: Settings → Pages → Source를 **None** 으로
  - (이미 `https://wltns166-boop.github.io/team-tops/` 로 떠 있을 수 있음)

## 1단계 — Firebase 인증(이메일/비밀번호) 켜기

- Firebase 콘솔 → **Authentication → 시작하기 → 로그인 방법 → 이메일/비밀번호 → 사용 설정**

## 2단계 — 계정 만들기 + 평문 비밀번호 제거 (스크립트)

1. 콘솔 → **프로젝트 설정 → 서비스 계정 → 새 비공개 키 생성** → 받은 파일을 이 폴더에 `serviceAccount.json` 으로 저장
2. 터미널에서:
   ```
   npm install firebase-admin
   node provision-users.js
   ```
3. 생성된 **passwords.csv** 의 임시 비밀번호를 팀원에게 전달 → 첫 로그인 후 **비밀번호 변경**(로그인 화면의 "비밀번호 변경") 안내
4. 전달 끝나면 **passwords.csv, serviceAccount.json 삭제**

## 3단계 — Firestore 규칙 게시 (← 실제 차단되는 순간)

- 콘솔 → **Firestore Database → 규칙(Rules)** 탭에 `firestore.rules` 내용을 붙여넣고 **게시**
- 이때부터 로그인 안 한 접근은 전부 막힙니다.

## 4단계 — 새 앱 적용 + 로그인 테스트

- 이 브랜치(`claude/zen-goldberg-YNnA8`)의 `insurance-intranet-v2.html` 로 교체
- **반드시 테스트**: 관리자/팀원 각각 로그인 → 데이터 보이는지 → 비밀번호 변경 동작하는지
- 잘 되면 운영 파일 교체

## 5단계 — 마무리

- 관리자 비밀번호(기존 `3399`), 팀원 비밀번호(기존 `0000`)는 유출됐으니 **모두 교체됨**(2단계에서 임시값으로 바뀜). 전원 변경 확인.
- (권장) Firebase **API 키 제한**: 콘솔 → 프로젝트 설정 → 앱 → API 키 → HTTP 리퍼러 제한
- 백업/복원, 클라우드 업로드 기능도 이제 로그인 후에만 동작합니다.

---

## 바뀐 점 (코드)

- 로그인: 코드에 박아둔 비밀번호 비교 → **Firebase 인증**으로 변경
- 관리자 비밀번호(`3399`)·기본 비밀번호(`0000`) → **코드에서 삭제**
- "비밀번호 찾기": 평문을 그대로 보여주던 부분 → **표시 안 함**(관리자 초기화 안내)
- "비밀번호 변경": Firebase 인증으로 처리(최소 **6자**)
- 팀원 수정 화면에서는 비밀번호를 직접 못 바꿈 → 초기화는 위 스크립트로
