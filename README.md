# InsureNet 인트라넷 — 핸드폰 연동

PC와 휴대폰이 **같은 데이터를 실시간으로 공유**하는 보험 영업 인트라넷입니다.
프론트엔드(`insurance-intranet-v2.html`)는 그대로 두고, 뒤에 **백엔드 서버**를 붙여
여러 기기가 같은 고객·청구·DB·메신저 데이터를 보도록 만들었습니다.
휴대폰 홈 화면에 **앱처럼 설치(PWA)**할 수도 있습니다.

---

## 구조

```
┌────────────┐        ┌──────────────────────┐
│  PC 브라우저  │◀──────▶│                      │
└────────────┘  HTTP  │   InsureNet 서버       │
┌────────────┐  /api  │  (Node + Express)     │
│  폰 브라우저   │◀──────▶│  - 로그인 인증(JWT)     │
│  /설치한 앱   │        │  - 공유 데이터 저장소     │
└────────────┘        │  - 프론트엔드 서빙        │
                      └──────────────────────┘
                              │
                       server/data/db.json  (데이터 영속 저장)
```

- **로그인**: 서버가 인증하고 토큰(JWT)을 발급합니다. 비밀번호는 해시로 저장됩니다.
- **데이터 동기화**: 화면에서 무언가 저장하면 즉시 서버로 올라가고(`PUT /api/state/...`),
  다른 기기는 4초마다 변경을 감지해 자동으로 화면을 갱신합니다.
- **오프라인**: 서버에 연결할 수 없으면 기존처럼 그 기기 안에서만(localStorage) 동작합니다.

## 실행 방법

### 1) 서버 켜기 (PC에서 1회)

```bash
cd server
npm install            # 최초 1회만
JWT_SECRET=원하는비밀문자열 npm start
```

실행되면 이렇게 표시됩니다:

```
InsureNet 서버 실행 중
 ─ 로컬:   http://localhost:4000
```

> `JWT_SECRET`은 로그인 토큰 서명에 쓰이는 비밀 키입니다.
> 지정하지 않으면 임시 키가 생기고, 서버를 재시작할 때마다 로그인이 풀립니다.
> 운영할 때는 꼭 지정하세요.

### 2) PC에서 접속

브라우저에서 **http://localhost:4000** 접속 → 로그인.

기본 계정:

| 아이디 | 비밀번호    | 역할  |
|--------|-----------|------|
| admin  | admin1234 | 관리자 |
| fc1    | fc1234    | FC   |
| fc2    | fc2234    | FC   |

### 3) 휴대폰에서 접속 (같은 와이파이)

휴대폰이 PC와 **같은 와이파이**에 있어야 합니다.

1. PC의 내부 IP를 확인합니다.
   - Windows: `ipconfig` → "IPv4 주소" (예: `192.168.0.10`)
   - Mac/Linux: `ifconfig` 또는 `ip addr` → `192.168.x.x`
2. 휴대폰 브라우저에서 **http://<PC의 IP>:4000** 접속 (예: `http://192.168.0.10:4000`)
3. 로그인하면 PC와 **같은 데이터**가 보입니다. 폰에서 고객을 추가하면 PC에도 바로 반영됩니다.

> PC 방화벽이 4000번 포트를 막을 수 있습니다. 접속이 안 되면 해당 포트를 허용하세요.

### 4) 폰 홈 화면에 앱으로 설치 (PWA)

- **Android (Chrome)**: 접속 후 메뉴(⋮) → **"홈 화면에 추가"**
- **iPhone (Safari)**: 공유 버튼 → **"홈 화면에 추가"**

설치하면 주소창 없이 앱처럼 전체화면으로 실행되고, 한 번 로그인하면 토큰이 저장되어
다음부터는 자동 로그인됩니다.

> 참고: 폰 설치형 앱의 오프라인 캐시(서비스워커)는 보안상 **HTTPS 또는 localhost**에서만
> 동작합니다. 사내망 IP(http)에서도 설치·사용은 되지만 완전한 오프라인 캐시를 원하면
> 아래 "외부/HTTPS 공개"를 참고하세요.

## 회사 밖에서도 접속하기 (클라우드 배포)

서버를 클라우드에 올리면 **HTTPS 주소 하나로 어디서든** 접속·설치할 수 있고,
폰 홈 화면 PWA의 완전한 오프라인 캐시도 동작합니다. 두 가지 방법을 준비해 두었습니다.

### 방법 A — Render (가장 쉬움, 무료 플랜)

이 저장소에 `render.yaml`이 포함되어 있어 클릭 몇 번이면 됩니다.

1. <https://render.com> 가입 후 **New ▸ Blueprint**
2. 이 GitHub 저장소 선택 → Render가 `render.yaml`을 읽어 자동 구성
   - `JWT_SECRET` 자동 생성, 영속 디스크(`/var/data`)에 데이터 저장
3. 배포가 끝나면 `https://insurenet-xxxx.onrender.com` 같은 주소가 생깁니다.
   PC·폰 어디서든 이 주소로 접속하면 같은 데이터를 공유합니다.

> 무료 플랜은 일정 시간 미사용 시 잠들었다 첫 요청에 깨어나느라 수십 초 걸릴 수 있습니다.
> 상시 가동이 필요하면 유료 플랜을 쓰세요.

### 방법 B — Docker (Railway, Fly.io, Cloud Run, 사내 서버 등 어디서나)

`Dockerfile`이 포함되어 있습니다.

```bash
docker build -t insurenet .
docker run -d -p 4000:4000 \
  -e JWT_SECRET=원하는비밀문자열 \
  -v insurenet-data:/data \
  insurenet
```

- `-v insurenet-data:/data` : 데이터를 볼륨에 영속 저장 (컨테이너를 지워도 유지)
- Railway/Fly.io 등은 이 `Dockerfile`을 그대로 감지해 배포합니다.
  영속 저장을 위해 볼륨을 `/data`에 마운트하고, 환경변수 `JWT_SECRET`을 설정하세요.

### 배포 시 꼭 설정할 환경변수

| 변수 | 설명 |
|------|------|
| `JWT_SECRET` | 로그인 토큰 서명 키. **반드시 지정** (없으면 재시작마다 로그인 풀림) |
| `DATA_DIR` | 데이터(`db.json`) 저장 폴더. 영속 디스크/볼륨 경로로 지정 |
| `PORT` | 대개 플랫폼이 자동 주입. 미지정 시 4000 |

> 서버는 종료 신호(재배포 시 SIGTERM)를 받으면 저장 대기 중인 변경을 디스크에 마저 기록한 뒤
> 종료하므로, 재배포해도 직전 데이터가 유실되지 않습니다.

## 그다음 단계 (선택)

| 원하는 것 | 방법 |
|----------|------|
| 앱스토어/플레이스토어 앱 | [Capacitor](https://capacitorjs.com)로 이 웹앱을 감싸 APK/IPA 빌드 |
| 푸시 알림 | 백엔드에 웹푸시 또는 FCM 연동 추가 |

프론트엔드가 (다른 주소의) 서버에 붙도록 강제하려면, 브라우저 콘솔에서:

```js
localStorage.setItem('in_apiBase', 'https://your-server.example.com');
```

같은 서버가 프론트엔드도 서빙하므로, 보통은 배포 주소로 접속만 하면 자동으로 맞춰집니다.

## API 요약

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/login` | 로그인 → `{ token, user }` |
| GET  | `/api/me` | 토큰으로 내 정보 확인 |
| GET  | `/api/state` | 전체 공유 데이터 조회 |
| GET  | `/api/state/version` | 변경 감지용 버전(폴링) |
| PUT  | `/api/state/:key` | 한 항목 저장 (customers, claims, chats 등) |
| PUT  | `/api/state` | 여러 항목 일괄 저장 |
| GET  | `/api/health` | 서버 상태 확인 |

`/api/state` 계열은 모두 `Authorization: Bearer <token>` 헤더가 필요합니다.
