# InsureNet v2 — 보험 영업 인트라넷 데모

보험 영업 조직을 위한 사내 인트라넷 데모입니다.  
서버 없이 브라우저 하나로 바로 실행되는 단일 HTML 파일로, 모든 데이터는 브라우저의 localStorage에 저장됩니다.

---

## 실행 방법

별도 설치나 서버 구동이 필요 없습니다.

```
insurance-intranet-v2.html 파일을 브라우저(Chrome, Edge, Firefox 등)로 열기
```

파일을 열면 로그인 화면이 표시됩니다. 아래 계정으로 접속할 수 있습니다.

| 구분 | 아이디 | 비밀번호 |
|------|--------|----------|
| 관리자 | `admin` | `admin1234` |
| FC (영업사원) | `fc1` | `fc1234` |
| FC (영업사원) | `fc2` | `fc2234` |

관리자 계정으로 로그인하면 DB 배정, 관리자 패널 등 추가 메뉴가 활성화됩니다.

---

## 주요 기능

### 대시보드
로그인 후 처음 보이는 화면입니다. 이번 달 청구 건수, 처리 대기 건수, DB 배정 현황, 월 목표 달성률을 한눈에 볼 수 있습니다. 최근 공지사항과 최근 보험금 청구 내역도 함께 표시됩니다.

### 공지사항
전체 공지, 중요 공지, 팀 공지 탭으로 나뉩니다. 관리자 패널에서 새 공지를 등록하면 이 목록에 바로 반영됩니다. 인쇄 및 PDF 출력을 지원합니다.

### 일정/캘린더
2025년 5월 기준 월간 달력이 표시됩니다. "일정 추가" 버튼을 눌러 날짜, 제목, 유형(팀미팅·교육·마감)을 입력하면 해당 날짜 칸에 일정이 표시됩니다. 추가한 일정은 localStorage에 저장됩니다.

### 문서 자료실
약관/규정, 교육자료, 양식, 보고서 카테고리로 분류된 공유 문서 목록입니다. 파일명이나 작성자로 검색하거나 태그를 눌러 필터링할 수 있습니다. 문서 목록은 코드 내에 고정된 샘플 데이터로 구성되어 있습니다(다운로드 버튼은 토스트 알림만 표시하는 데모 동작).

### 직원 디렉토리
임직원 카드 목록과 SVG 기반 조직도를 함께 제공합니다. 이름·부서·직책으로 검색이 가능하며, 신규 직원 등록 및 삭제를 지원합니다. 조직도는 편집 모드에서 노드를 클릭하여 이름, 담당자, 상위 노드, 색상 테마를 수정하거나 새 노드를 추가할 수 있습니다.

### 고객정보 정리
고객의 기본 정보(이름, 생년월일, 성별, 연락처, 이메일, 직업, 주소, 담당 FC, 메모)를 등록하고 테이블로 관리합니다. PDF 출력 및 인쇄를 지원합니다.

### 병력 정리
고객별 병력 사항(질병명, 진단일, 치료 여부, 의료기관, 심사 영향도)을 등록하고 관리합니다. 고객명으로 검색할 수 있으며 심사 영향도(낮음·보통·높음·인수불가)는 색상 태그로 구분됩니다.

### 보장분석표
가입제안서 파일을 기반으로 고객의 보장 현황을 분석하고 시각화합니다. 자세한 사용법은 아래 섹션을 참고하세요.

### 보험금 청구
보험금 청구를 접수하고 상태(접수·심사중·완료·반려)를 관리합니다. 청구 상태는 테이블의 드롭다운으로 바로 변경할 수 있으며, 변경 시 알림이 발생합니다. 접수 대기·심사중·완료·반려 건수가 상단 통계 카드에 실시간으로 집계됩니다.

### DB 신청
영업 DB(고객 리드)를 신청합니다. DB 유형, 대상 연령대, 신청 수량, 희망 지역, 관심 상품군을 입력하면 신청 번호가 자동으로 부여됩니다. 신청 내역은 테이블로 확인할 수 있습니다.

### DB 배정 (관리자 전용)
관리자 계정에서만 보이는 메뉴입니다. 대기 중인 DB를 FC에게 배정합니다. 드롭다운으로 담당 FC를 선택하고 "배정" 버튼을 누르면 상태가 배정완료로 바뀝니다.

### 관리자 패널 (관리자 전용)
- 공지사항 등록: 제목, 유형(일반·중요), 내용을 입력하여 새 공지를 등록합니다.
- 전체 데이터 내보내기: 고객, 병력, 청구, DB 신청 데이터를 JSON 파일로 다운로드합니다.
- 전체 초기화: localStorage에 저장된 모든 데이터를 삭제합니다(되돌릴 수 없으므로 주의).

### 전역 검색
상단 헤더의 검색창에 키워드를 입력하면 페이지 이름, 등록된 고객, 보험금 청구 내역을 동시에 검색합니다. 결과를 클릭하면 해당 페이지로 바로 이동합니다.

### 알림
헤더의 벨 아이콘을 클릭하면 알림 패널이 열립니다. 청구 상태 변경, 공지 등록, DB 배정 등 주요 동작 시 알림이 자동으로 생성됩니다. 개별 클릭 또는 "모두 읽음" 버튼으로 읽음 처리할 수 있습니다.

### 팀 채팅
화면 우측 하단의 채팅 버튼을 누르면 채팅 창이 열립니다. 전체, 영업1팀, 영업2팀, 관리팀 채널로 나뉘며 채널별 대화가 localStorage에 저장됩니다. Enter 키로 메시지를 보낼 수 있습니다(줄바꿈은 Shift+Enter).

## 보장분석표 사용법

보장분석표는 가입제안서 파일을 업로드하면 항목별 보장 현황을 자동으로 계산하여 표로 보여주는 기능입니다.

**단계별 사용법**

1. 사이드바에서 "보장분석표" 메뉴를 클릭합니다.
2. "보장분석 작성" 카드의 "고객명" 입력란에 분석할 고객의 이름을 입력합니다.
3. "가입제안서 업로드" 파일 선택 버튼을 클릭하여 제안서 파일을 선택합니다.  
   (PDF, PNG, JPG, JPEG, XLSX, XLS 형식을 지원합니다)
4. 파일 정보가 파란색으로 표시되면 선택이 완료된 것입니다.
5. "보장분석표 작성" 버튼을 클릭합니다.

작성이 완료되면 아래와 같은 분석 결과 테이블이 생성됩니다.

| 보장항목 | 현재 보장 | 권장 보장 | 부족액 | 충족률 | 상태 |
|----------|-----------|-----------|--------|--------|------|
| 사망보험금 | ... | 1억 원 | ... | ...% | 충족/부족/미가입 |
| 암 진단금 | ... | 3천만 원 | ... | ...% | ... |
| 실손의료비 | ... | 5천만 원 | ... | ...% | ... |
| 입원일당 | ... | 50,000원/일 | ... | ...% | ... |
| 뇌졸중/심근경색 | ... | 1천만 원 | ... | ...% | ... |

분석 결과 하단에는 부족한 보장 항목을 요약한 종합 의견이 자동으로 생성됩니다.

> 참고: 현재 보장 수치는 업로드된 파일의 이름과 크기를 시드(seed)로 사용하여 생성하는 데모 값입니다. 실제 제안서 내용을 파싱하지는 않습니다.

작성된 보장분석표는 저장되며 "고객명 검색"으로 필터링하거나 "PDF 출력" 버튼으로 인쇄할 수 있습니다.

## 데이터 저장 방식

모든 데이터는 별도의 서버나 데이터베이스 없이 브라우저의 **localStorage**에 저장됩니다.

저장 시 키 이름에 `in_` 접두사가 붙습니다. 저장되는 항목은 다음과 같습니다.

| localStorage 키 | 저장 내용 |
|-----------------|-----------|
| `in_customers` | 고객정보 목록 |
| `in_medicals` | 병력 목록 |
| `in_claims` | 보험금 청구 목록 |
| `in_dbreqs` | DB 신청 목록 |
| `in_notices` | 공지사항 목록 |
| `in_calEvents` | 캘린더 일정 |
| `in_chats` | 채팅 메시지 |
| `in_notifs` | 알림 목록 |
| `in_staff` | 직원 목록 |
| `in_orgNodes` | 조직도 노드 |
| `in_coverages` | 보장분석표 목록 |

처음 실행 시 샘플 데이터가 자동으로 채워집니다. 이후에는 저장된 데이터가 그대로 유지됩니다.

**데이터 초기화 방법**

- 관리자 패널 > "전체 초기화" 버튼: 주요 데이터 키를 한 번에 삭제합니다(삭제 후 새로고침 필요).
- 브라우저 개발자 도구 콘솔에서 직접 삭제할 수도 있습니다.

```js
// 특정 항목만 삭제
localStorage.removeItem('in_customers');

// in_ 접두사 항목 전체 삭제
Object.keys(localStorage).filter(k => k.startsWith('in_')).forEach(k => localStorage.removeItem(k));
```

> 주의: localStorage는 브라우저 탭·창을 닫아도 유지되지만, 브라우저 데이터를 지우거나 다른 브라우저에서 열면 데이터가 없습니다. 중요한 데이터는 관리자 패널의 "전체 데이터 내보내기" 기능으로 JSON 백업을 만들어 두세요.

## 기술 스택

| 항목 | 내용 |
|------|------|
| 구조 | 순수 HTML / CSS / JavaScript (단일 파일 SPA) |
| 프레임워크 | 없음 (바닐라 JS, 외부 JS 라이브러리 미사용) |
| 외부 리소스 | Google Fonts (`Noto Sans KR`, `DM Mono`) — 인터넷 연결 없으면 폴백 폰트 사용 |
| 데이터 저장 | 브라우저 localStorage |
| 백엔드 | 없음 |
| 빌드 도구 | 없음 (파일 그대로 실행) |

Google Fonts 외에 외부 CDN이나 npm 패키지에 대한 의존성이 없습니다. 인터넷이 끊긴 환경에서도 폰트만 시스템 기본 폰트로 대체될 뿐 모든 기능은 정상 동작합니다.
