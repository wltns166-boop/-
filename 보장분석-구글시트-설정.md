# 보장분석표 — 구글 스프레드시트 방식 (설정 안내)

보장분석표를 **구글 스프레드시트로 작업**하고 **엑셀(.xlsx)로 저장**합니다.
양식을 **구글시트로 직접 만들어 등록**하므로, 색·테두리·병합·수식이 **100% 그대로** 유지됩니다.
(엑셀 변환 단계가 없어 더 안정적입니다.)

## 동작 방식
1. 관리자가 **구글시트로 양식을 만들고**, 인트라넷에서 **그 시트 주소(URL)를 한 번 등록**.
2. 팀원이 **고객명 + PDF + [작성]** → AI가 보장내용을 읽고,
   서버가 **양식을 복사**해 고객별 시트에 **값만 기입** → 화면에 그대로 임베드.
3. **[엑셀로 저장]** → .xlsx 다운로드 / **[구글시트 열기]** → 새 탭 편집.
4. 저장 위치: `…/{팀원}/보장분석표/{고객명}/{고객명}님 보장분석표`

---

## ⚙️ 한 번만 해두면 되는 설정

### 1) 보장분석 양식을 구글시트로 만들어 두기
- 사장님 구글 드라이브(서버가 쓰는 그 계정)에서 **구글 스프레드시트로 양식 제작**.
- **탭(시트) 순서**: ①전 → ②후 → ③비교 (1·2·3번 순서대로).
- 이미 엑셀 양식이 있으면: 구글시트로 열어 **사본을 구글시트로 저장**하면 됩니다.
- (다른 사람이 만들었다면, 그 시트를 **사장님 계정에 공유**해 두세요.)

### 2) 앱스크립트(google-drive-sync.gs) 재배포  ← 서버 코드 변경
1. https://script.google.com → 프로젝트 열기
2. `google-drive-sync.gs` **전체 복사 → 붙여넣기 → 저장(💾)**
3. **배포 → 배포 관리 → ✏️ → 새 버전 → 배포**
4. **권한 승인 창이 다시 뜨면 허용** (시트 내보내기에 외부요청 권한이 필요 — 1회)

> 확인: `웹앱URL?ping=1` → `{"ok":true,"version":"gsheet-2"}` 면 최신.

### 3) Functions(/api/drive) 재배포  ← 브라우저가 시트 주소를 읽으려면 필요
브라우저는 보안정책(CORS)으로 앱스크립트 응답을 직접 못 읽습니다.
같은 도메인의 함수(`/api/drive`)가 대신 호출하도록 했습니다.
```bash
firebase deploy --only functions
```
(AI 분석 `/api/analyze`와 같은 함수 — 이미 쓰고 있으면 이 한 줄이면 됩니다. Blaze 요금제 필요.)

## 사용 시작
1. 인트라넷 **새로고침(Ctrl+Shift+R)**
2. 보장분석 → **[구글시트 양식 등록(관리자)]** → 1)에서 만든 양식 **URL 붙여넣기**
   → "구글시트 양식이 등록되었습니다" 뜨면 OK
3. **고객명 + PDF + [작성]** → 구글시트가 화면에 임베드되는지 확인

## 참고 (보안)
- 만든 고객 보장분석표는 임베드 편집을 위해 **링크 공유(보기·편집)** 로 열립니다.
- 더 엄격히(보기 전용/특정 계정만) 원하면 말씀해 주세요. 서버 한 줄(`Permission.VIEW` 등)로 조정합니다.
