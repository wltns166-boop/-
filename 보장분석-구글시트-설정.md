# 보장분석표 — 구글 스프레드시트 방식 (설정 안내)

보장분석표를 **구글 스프레드시트로 작업**하고 **엑셀(.xlsx)로 저장**하도록 바꿨습니다.
양식의 색·테두리·병합·수식이 **그대로 유지**됩니다. (엑셀→브라우저 변환에서 서식이 깨지던 문제 해결)

## 동작 방식
1. 관리자가 **[엑셀 양식 업로드]** → 서버가 그 엑셀을 **구글시트 양식**으로 자동 변환·등록.
2. 팀원이 **고객명 입력 + PDF 업로드 + [작성]** → AI가 보장내용을 읽어,
   서버가 양식을 복사해 **고객별 구글시트**에 값만 기입 → 화면에 **그대로 임베드**.
3. **[엑셀로 저장]** → 구글시트를 .xlsx로 변환해 다운로드.
   **[구글시트 열기]** → 새 탭에서 직접 편집.
4. 저장 위치: 드라이브 `…/{팀원}/보장분석표/{고객명}/{고객명}님 보장분석표`

---

## ⚙️ 한 번만 해두면 되는 설정 (2가지)

### 1) 앱스크립트(google-drive-sync.gs) 재배포  ← 서버 코드가 바뀜
1. https://script.google.com 에서 프로젝트 열기
2. 저장소의 `google-drive-sync.gs` **전체 복사 → 붙여넣기 → 저장(💾)**
3. **배포 → 배포 관리 → ✏️ → 새 버전 → 배포**
4. 처음 실행 시 **권한 승인 창**이 다시 뜹니다 → 본인 구글계정으로 **허용**
   (새 기능이 외부요청·드라이브 변환 권한을 쓰기 때문 — 1회만)

> 확인: 브라우저에서 `웹앱URL?ping=1` → `{"ok":true,"version":"gsheet-1"}` 면 최신.

### 2) Functions(/api/drive) 재배포  ← 브라우저가 시트 주소를 읽으려면 필요
브라우저는 보안정책(CORS) 때문에 앱스크립트 응답을 직접 못 읽습니다.
그래서 같은 도메인의 함수(`/api/drive`)가 대신 호출하도록 했습니다. (AI 분석 `/api/analyze`와 같은 함수)

```bash
firebase deploy --only functions
```
(이미 AI 분석 기능을 쓰고 있다면 같은 방식으로 한 줄이면 됩니다. Blaze 요금제 필요.)

## 설정 전/후
- **설정 전**: [작성]을 누르면 "구글시트 생성 실패" 또는 "서버 오류"가 납니다.
- **설정 후**: 정상 작동. 양식만 한 번 다시 [엑셀 양식 업로드]로 등록하면 구글시트 양식으로 변환됩니다.

## 참고 (보안)
- 만든 고객 보장분석표는 **링크가 있으면 팀원이 보고 편집**할 수 있도록 공유됩니다(임베드 편집을 위해).
- 더 엄격하게(보기 전용 / 특정 계정만)으로 바꾸고 싶으면 말씀해 주세요. 서버 한 줄로 조정합니다.
