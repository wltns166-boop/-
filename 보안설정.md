# 🔒 청구서류 보안 설정 방법 (인증 연동)

청구서류에는 **고객 주민등록번호·신분증·통장 사본** 같은 민감 개인정보가 들어갑니다.
이걸 외부에 노출하지 않으면서 팀원끼리만 공유하도록, **Firebase 인증을 켜고
Storage·Firestore 접근을 "로그인한 세션만 허용"으로 잠그는** 작업입니다.

> 앱 코드는 이미 수정해서, 페이지가 열릴 때 자동으로 Firebase 익명 인증에 로그인합니다.
> 아래 콘솔 설정만 해주시면 보안이 적용됩니다.

---

## ⚠️ 순서가 중요합니다 (안 지키면 앱이 안 열릴 수 있어요)

```
[1] Firebase에서 "익명 인증" 켜기   ← 이걸 먼저! (안 하면 아래 규칙 적용 시 앱 전체가 막힘)
        ↓
[2] 새 버전 배포되었는지 확인 (자동)
        ↓
[3] 앱이 정상 동작하는지 테스트
        ↓
[4] Storage 규칙 적용
        ↓
[5] Firestore 규칙 적용
        ↓
[6] 다시 테스트
```

---

## 1단계 — 익명 인증 켜기 (가장 먼저, 필수)

1. https://console.firebase.google.com → **team-tops-intranet** 선택
2. 왼쪽 메뉴 **빌드(Build)** → **Authentication** 클릭
3. **시작하기**(Get started) 버튼이 보이면 클릭
4. 위쪽 **Sign-in method** 탭 클릭
5. 공급업체 목록에서 **익명**(Anonymous) 클릭 → **사용 설정(Enable)** 토글 켜기 → **저장**

✔️ "익명"이 "사용 설정됨" 상태가 되면 OK.

---

## 2단계 — 새 버전 배포 확인

코드는 이미 푸시되어 자동 배포됩니다. GitHub **Actions** 탭에서 최신 배포가
초록 체크(✓)인지 확인하고, https://team-tops-intranet.web.app 를 새로고침하세요.

---

## 3단계 — 정상 동작 테스트 (규칙 적용 전)

규칙을 잠그기 **전에**, 익명 인증이 잘 붙는지 먼저 확인합니다.

1. https://team-tops-intranet.web.app 접속 → 로그인
2. 데이터(공지·팀원 등)가 잘 보이면 OK
3. (선택) 브라우저 F12 → Console 탭에 `✅ Firebase 연결 성공` 이 보이고
   빨간 인증 오류가 없으면 정상

---

## 4단계 — Storage 규칙 적용

1. Firebase 콘솔 → 왼쪽 **빌드** → **Storage** → 위쪽 **Rules(규칙)** 탭
2. 기존 내용을 모두 지우고, 저장소의 **`storage.rules`** 파일 내용을 그대로 붙여넣기
   (또는 아래 내용)
3. **게시(Publish)** 클릭

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /claim_templates/{allPaths=**} { allow read, write: if request.auth != null; }
    match /wa_templates/{allPaths=**}    { allow read, write: if request.auth != null; }
    match /claim_packages/{allPaths=**}  { allow read, write: if request.auth != null; }
    match /{allPaths=**}                 { allow read, write: if false; }
  }
}
```

---

## 5단계 — Firestore 규칙 적용

1. Firebase 콘솔 → 왼쪽 **빌드** → **Firestore Database** → 위쪽 **규칙(Rules)** 탭
2. 기존 내용을 지우고, 저장소의 **`firestore.rules`** 내용을 붙여넣기 (또는 아래)
3. **게시(Publish)** 클릭

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} { allow read, write: if request.auth != null; }
  }
}
```

---

## 6단계 — 다시 테스트

https://team-tops-intranet.web.app 새로고침 → 로그인 → 데이터가 잘 보이고,
청구신청·청구파일 업로드/다운로드가 되는지 확인.

### ❗ 만약 데이터가 안 보이거나 오류가 나면 (롤백)
대부분 **1단계(익명 인증)를 안 켰을 때** 생깁니다. 그럴 땐:
- 4·5단계 규칙을 잠시 `allow read, write: if true;` 로 되돌리고(게시),
- 1단계 익명 인증을 켠 뒤,
- 다시 4·5단계 규칙을 적용하세요.

---

## 📌 이 보안의 수준 (꼭 알아두세요)

- ✅ **막아주는 것:** 인증 없이 URL·버킷에 직접 접근, 검색엔진 크롤링, 외부의
  우연한/캐주얼한 접근, 데이터 무단 열람.
- ⚠️ **한계:** 이 앱은 공개된 API 키로 동작하고 익명 인증을 쓰기 때문에,
  **작정한 공격자**는 익명 토큰을 스스로 발급해 접근을 시도할 여지가 있습니다.
- 🔐 **더 강한 보안(다음 단계):** 팀원 한 명 한 명에게 **개별 계정(이메일/비번)** 을
  발급하고 **가입은 관리자만** 할 수 있게 막는 방식. 주민번호급 정보를 다룬다면
  이 단계까지 가는 것을 권장합니다. 준비되시면 도와드리겠습니다.
