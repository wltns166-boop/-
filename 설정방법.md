# 🚀 자동 배포 설정 방법 (그대로 따라하기)

> 이 작업은 **처음 딱 한 번만** 하면 됩니다.
> 끝내고 나면, 앞으로는 페이지를 수정할 때마다 팀원 전체에게 **자동으로** 반영됩니다.
> 컴퓨터(노트북/PC)에서 하시는 걸 권장합니다. 천천히 따라오세요. 약 10분 걸립니다.

배포되는 주소(설정 완료 후): **https://team-tops-intranet.web.app**

---

## ✅ 전체 흐름 (지도)

```
[1] Firebase에서 "열쇠 파일(JSON)" 받기
        ↓
[2] 그 열쇠를 GitHub "금고(Secret)"에 넣기
        ↓
[3] 끝! 이제 수정 → 자동 배포
        ↓
[4] (선택) AI 보장분석 기능까지 켜기
```

---

## 📌 1단계 — Firebase에서 열쇠 파일 받기

GitHub이 내 Firebase에 배포하려면 "허락받은 열쇠"가 필요합니다.

1. 인터넷 주소창에 입력 → 접속:  **https://console.firebase.google.com**
2. 프로젝트 목록에서 **`team-tops-intranet`** 클릭
3. 왼쪽 위 **톱니바퀴(⚙️) 아이콘** 클릭 → **프로젝트 설정** 클릭
4. 위쪽 탭 중 **서비스 계정** (Service accounts) 클릭
5. 화면 아래쪽 **`새 비공개 키 생성`** (Generate new private key) 파란 버튼 클릭
6. 경고창이 뜨면 **`키 생성`** 클릭
7. → **JSON 파일이 자동으로 다운로드**됩니다. (예: `team-tops-intranet-xxxx.json`)

> ⚠️ 이 파일은 **비밀번호와 같습니다.** 카톡·메일로 남에게 보내지 마세요.

✔️ **체크포인트:** 다운로드 폴더에 `.json` 파일이 생겼나요? 됐으면 2단계로.

---

## 📌 2단계 — GitHub 금고에 열쇠 넣기

1. 인터넷에서 내 GitHub 저장소 페이지로 이동
   → **https://github.com/wltns166-boop/-**
2. 위쪽 메뉴 맨 오른쪽 **`Settings`** (설정, 톱니바퀴 모양) 클릭
3. 왼쪽 목록을 아래로 내려서 **`Secrets and variables`** 클릭 → 그 아래 **`Actions`** 클릭
4. 오른쪽 위 초록색 **`New repository secret`** 버튼 클릭
5. 칸을 아래처럼 채웁니다:
   - **Name (이름)** 칸:  `FIREBASE_SERVICE_ACCOUNT`
     - ※ 띄어쓰기·오타 없이 **정확히 그대로** 입력 (대문자, 밑줄 `_` 주의)
   - **Secret (값)** 칸:  1단계에서 받은 **JSON 파일을 메모장으로 열어서**,
     안의 **모든 글자를 전체 선택(Ctrl+A) → 복사(Ctrl+C) → 여기에 붙여넣기(Ctrl+V)**
     - ※ `{` 부터 `}` 까지 통째로 다 넣어야 합니다.
6. 아래 초록색 **`Add secret`** 버튼 클릭

✔️ **체크포인트:** 목록에 `FIREBASE_SERVICE_ACCOUNT` 가 보이면 성공!

---

## 📌 3단계 — 끝! 자동 배포 확인하기

이제 끝났습니다. 작동하는지 직접 한번 돌려봅시다.

1. GitHub 저장소 위쪽 메뉴에서 **`Actions`** 탭 클릭
2. 왼쪽에서 **`Deploy to Firebase`** 클릭
3. 오른쪽 **`Run workflow`** 버튼 클릭 → 다시 한번 **`Run workflow`** 클릭
4. 잠시 기다리면(2~4분) 줄에 **초록색 체크(✓)** 가 뜹니다.
5. 브라우저에서 **https://team-tops-intranet.web.app** 접속 → 인트라넷이 뜨면 성공! 🎉

> 앞으로는 따로 버튼 누를 필요 없습니다.
> **페이지(코드)를 수정해서 저장(푸시)하면 자동으로 위 주소에 반영**됩니다.
> 팀원들에게는 파일을 보낼 필요 없이 **이 주소 링크만 공유**하면 됩니다.

❓ 빨간색 X가 떴다면 → Actions에서 그 줄을 클릭하면 이유가 나옵니다.
   보통 2단계의 이름(`FIREBASE_SERVICE_ACCOUNT`) 오타이거나 값이 일부만 붙여넣어진 경우입니다.

## 📌 4단계 — (선택) AI 보장분석 기능까지 켜기

> 이건 **안 해도 됩니다.** 나머지 기능은 다 정상 작동하고, 이걸 안 하면
> "보장분석 AI 자동 분석"만 동작하지 않습니다. 나중에 해도 됩니다.

AI 분석은 Claude(AI)를 호출하는 서버가 필요해서 두 가지가 더 필요합니다.

### (1) Firebase를 Blaze 요금제로 변경
- Firebase 콘솔 → 왼쪽 아래 **`요금제 업그레이드`** → **Blaze(종량제)** 선택 → 카드 등록
- ※ 소규모 사용은 거의 무료 수준이지만, 카드 등록은 필수입니다.

### (2) AI 키 등록 (컴퓨터에서 한 번)
컴퓨터의 **명령 프롬프트(cmd)** 또는 **터미널**에 차례대로 입력:
```bash
npm install -g firebase-tools
firebase login
firebase functions:secrets:set ANTHROPIC_API_KEY
```
- 마지막 줄 실행 후 키를 물어보면, **`sk-ant-`** 로 시작하는 Claude API 키를 붙여넣기
- 이후 페이지를 수정해 푸시하면 AI 서버까지 자동 배포됩니다.
