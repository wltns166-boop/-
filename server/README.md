# DB 배정 알림톡 서버 (솔라피)

인트라넷에서 DB를 배정하면 배정된 팀원(FC)에게 **카카오 알림톡**을 자동 발송하는 서버입니다.

```
인트라넷(HTML) ──POST /api/kakao/send──▶ 이 서버 ──▶ 솔라피 ──▶ 카카오 알림톡 ──▶ 팀원
```

---

## 1. 사전 준비 (한 번만)

1. **솔라피 가입** → <https://solapi.com>
2. **카카오 비즈니스 채널 연동**
   - 솔라피 콘솔 > 카카오 > 카카오톡 채널 연동
   - 이미 만든 **비즈니스 채널**을 검색해서 등록 → `발신 프로필 ID(pfId, KA01PF...)` 발급
3. **발신번호 등록·인증** (알림톡 실패 시 문자 대체발송용)
4. **템플릿 등록 + 승인 신청** (아래 3번 참고) → 승인되면 `템플릿 ID(KA01TP...)` 발급
5. **API Key 발급**: 솔라피 콘솔 > 개발/연동 > API Key 관리

## 2. 설치 & 실행

```bash
cd server
npm install
cp .env.example .env      # .env 를 열어 발급받은 값들을 채워 넣기
npm start                 # http://localhost:3000 에서 실행
```

## 3. 등록할 알림톡 템플릿 문구

솔라피 콘솔에서 아래 내용으로 템플릿을 등록하고 **승인 신청**하세요. (심사 보통 1~2영업일)
변수는 `#{변수명}` 형식이며, 서버가 보내는 변수명과 정확히 일치해야 합니다.

```
[DB 배정 알림]
#{이름}님, 새 DB가 배정되었습니다.

DB번호: #{디비번호}
고객명: #{고객명}
유형: #{유형}
지역: #{지역}

인트라넷에서 확인해 주세요.
```

> ⚠️ **개인정보 주의**: 고객 연락처·주민번호 등 민감정보를 템플릿에 넣으면 심사에서 반려될 수 있어
> 일부러 빼두었습니다. 상세 정보는 팀원이 인트라넷에서 확인하도록 유도하세요.
> 또한 알림톡은 **정보성 메시지만** 가능하며 광고성 문구는 불가합니다.

## 4. 인트라넷 연결

`insurance-intranet-v2.html` 의 `sendKakaoNotify()` 함수 위에 있는 설정값을 바꾸세요.

```js
const KAKAO_API_URL = 'http://localhost:3000/api/kakao/send'; // ← 이 서버 주소
```

값을 비워두면(`''`) 지금처럼 **시뮬레이션 모드**로 동작하고,
서버 주소를 넣으면 **실제 알림톡이 발송**됩니다.

## 5. 동작 확인 (서버만 단독 테스트)

```bash
curl -X POST http://localhost:3000/api/kakao/send \
  -H "Content-Type: application/json" \
  -d '{"to":"01012345678","variables":{"#{이름}":"김철수","#{디비번호}":"DB-1024","#{고객명}":"홍길동","#{유형}":"신규","#{지역}":"서울"}}'
```

- 알림톡: 약 9원/건 (+VAT) → 600건(30명×20건) 기준 약 5,400원
- 실패 시 문자 대체발송이 켜져 있으면 그 건만 SMS/LMS 요금(약 20~30원) 추가
