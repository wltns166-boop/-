# Apps Script(드라이브 서버) 배포 안내

## 먼저, 좋은 소식 — 이제 재배포는 거의 필요 없습니다
그동안 `google-drive-sync.gs` 를 자꾸 다시 배포해야 했던 이유는 **서버 코드 안에 폴더 구조가
박혀 있었기 때문**입니다. 지금은 서버를 **범용**으로 바꿨습니다.

- 서버는 이제 인트라넷이 보내는 **"이 경로에 이 파일을 저장하라"** 라는 지시만 받아 저장합니다.
- 그래서 **폴더 구조 변경 · 컬럼 추가 · 파일명 변경 · 새 분류**는 거의 다 `index.html`(클라이언트)만
  고치면 되고, 그건 **자동 배포(GitHub Actions)** 됩니다. → **`.gs` 재배포 불필요.**
- `.gs` 재배포가 필요한 경우는 "서버 동작 자체"가 바뀔 때뿐인데, 이제 거의 없습니다.

즉, **지금 한 번만 새 버전으로 배포해두면 앞으로는 신경 쓰지 않아도 됩니다.**

---

## 방법 A — 수동(가장 간단, 1~2분)
1. https://script.google.com 에서 프로젝트 열기
2. 저장소의 `google-drive-sync.gs` **전체 복사 → 붙여넣기 → 저장(💾)**
3. **배포 → 배포 관리 → ✏️(연필) → 새 버전 → 배포**
   (이렇게 하면 기존 `/exec` 주소가 그대로 유지됩니다.)

> 배포가 잘 됐는지 확인: 브라우저에서 `웹앱URL?ping=1` 열기 → `{"ok":true,"version":"generic-1"}` 나오면 최신.

---

## 방법 B — 한 줄 명령 자동 배포(고급, 선택)
완전 자동으로 하고 싶으면 `clasp`(구글 공식 CLI)로 **명령 한 번**에 배포할 수 있습니다.

### 처음 1회만 준비
1. `npx --yes @google/clasp login` — 브라우저에서 구글 로그인(1회)
2. https://script.google.com/home/usersettings → **"Apps Script API" 사용 ON**
3. 저장소 루트에 `.clasp.json` 만들기:
   ```json
   { "scriptId": "여기에_스크립트_ID" }
   ```
   (Apps Script 편집기 → 프로젝트 설정 ⚙️ → **스크립트 ID** 복사)
4. (선택) URL 유지하려면 배포 ID 저장:
   ```bash
   npx --yes @google/clasp deployments   # 목록에서 웹앱 배포 ID 확인
   echo "배포ID" > .gs-deployment-id
   ```

### 그 다음부터는 매번 이 한 줄
```bash
./deploy-gs.sh
```
코드 업로드 + 웹앱 배포까지 자동으로 끝납니다. (`.clasp.json`·`.gs-deployment-id`·`.clasprc.json`
은 비밀정보라 깃에 올라가지 않습니다.)

> ⚠️ 참고: 클라우드(웹)에서 도는 Claude 세션에는 **사장님 구글 로그인 정보가 없어서**, 제가 대신
> 구글 계정에 배포 버튼을 눌러드릴 수는 없습니다. 방법 B를 1회 세팅해 두면, 그 환경에서는
> `./deploy-gs.sh` 한 줄(또는 "gs 배포해줘")로 끝납니다.

## 배포 담당 에이전트
`apps-script-deployer` 에이전트를 만들어 두었습니다.
"**gs 배포해줘 / 앱스크립트 배포 / 재배포 해줘**" 라고 하면, 이 에이전트가
필요한 경우에만 `.gs` 를 고치고 `./deploy-gs.sh` 로 배포하며, 준비가 안 됐으면 무엇을 1회만
해두면 되는지 알려줍니다.
